# User Stories: Public Storefront

## Document Information

| Field | Value |
|---|---|
| Feature | Public Storefront |
| Actor | Visitor |
| Authentication | ไม่ต้องล็อกอิน |
| Status | Ready for specification |
| Optional scope | US-06 Sort by price |

## Epic

**ในฐานะ** ผู้เยี่ยมชมเว็บไซต์  
**ฉันต้องการ** เรียกดู ค้นหา และกรองสินค้าได้โดยไม่ต้องล็อกอิน  
**เพื่อ** สำรวจสินค้าและดูข้อมูลประกอบการตัดสินใจซื้อ

---

## US-01 — ดูรายการสินค้าทั้งหมด

**ในฐานะ** ผู้เยี่ยมชมที่ไม่ได้ล็อกอิน  
**ฉันต้องการ** เห็นรายการสินค้าพร้อมรูป ชื่อ และราคา  
**เพื่อ** จะได้รู้ว่ามีสินค้าอะไรขายบ้าง

### Acceptance Criteria

- [ ] แสดงการ์ดสินค้าแต่ละชิ้น โดยมีรูปหลัก 1 รูป ชื่อสินค้า และราคา
- [ ] ราคาแสดงในรูปแบบสกุลเงินที่อ่านง่าย
- [ ] สินค้าที่ไม่มีรูปต้องแสดง placeholder
- [ ] ผู้ใช้เข้าหน้ารายการสินค้าได้โดยไม่ต้องล็อกอิน
- [ ] ถ้าระบบยังไม่มีสินค้า ต้องแสดง empty state ที่เหมาะสม

---

## US-02 — กรองสินค้าตามหมวดหมู่

**ในฐานะ** ผู้เยี่ยมชม  
**ฉันต้องการ** กรองสินค้าตามหมวดหมู่  
**เพื่อ** หาสินค้าในกลุ่มที่สนใจได้เร็วขึ้น

### Acceptance Criteria

- [ ] แสดงรายการหมวดหมู่ทั้งหมด
- [ ] เมื่อเลือกหมวดหมู่ แสดงเฉพาะสินค้าที่อยู่ในหมวดหมู่นั้น
- [ ] มีตัวเลือก “ทั้งหมด” สำหรับล้างตัวกรอง
- [ ] หมวดหมู่ที่เลือกสะท้อนใน URL เพื่อให้แชร์ลิงก์ได้
- [ ] การเปลี่ยนหมวดหมู่ต้องไม่ทำให้คำค้นหาปัจจุบันหาย

---

## US-03 — ค้นหาสินค้าตามชื่อ

**ในฐานะ** ผู้เยี่ยมชม  
**ฉันต้องการ** ค้นหาสินค้าด้วยคำที่อยู่ในชื่อสินค้า  
**เพื่อ** หาสินค้าที่ต้องการได้ตรงขึ้น

### Acceptance Criteria

- [ ] ค้นหาจากชื่อสินค้าแบบไม่สนตัวพิมพ์เล็กหรือใหญ่
- [ ] รองรับการค้นหาแบบบางส่วนของชื่อสินค้า
- [ ] ถ้าค้นไม่พบ ต้องแสดง empty state
- [ ] การค้นหาต้องทำงานร่วมกับ category filter
- [ ] คำค้นหาปัจจุบันสะท้อนใน URL เพื่อให้แชร์ลิงก์ได้

---

## US-04 — ดูรายละเอียดสินค้า

**ในฐานะ** ผู้เยี่ยมชม  
**ฉันต้องการ** ดูรายละเอียดเต็มของสินค้าหนึ่งชิ้น  
**เพื่อ** ใช้ข้อมูลประกอบการตัดสินใจซื้อ

### Acceptance Criteria

- [ ] แสดงชื่อ คำอธิบาย ราคา และรูปสินค้าทั้งหมด
- [ ] แสดงชื่อหมวดหมู่ของสินค้า
- [ ] หน้ารายละเอียดใช้ URL รูปแบบ `/products/[id]`
- [ ] ถ้าไม่พบสินค้าตาม `id` ต้องแสดงหน้า 404
- [ ] ผู้ใช้เข้าหน้ารายละเอียดได้โดยไม่ต้องล็อกอิน

---

## US-05 — แบ่งหน้ารายการสินค้า

**ในฐานะ** ผู้เยี่ยมชม  
**ฉันต้องการ** ให้รายการสินค้าถูกแบ่งหน้า  
**เพื่อ** ให้หน้าโหลดได้เร็วแม้ระบบมีสินค้าจำนวนมาก

### Acceptance Criteria

- [ ] ระบบแสดงสินค้าตามจำนวนต่อหน้าที่กำหนดใน Specification
- [ ] มีตัวควบคุมสำหรับไปหน้าก่อนหน้าและหน้าถัดไป
- [ ] หน้าปัจจุบันสะท้อนใน URL
- [ ] การเปลี่ยนหน้าต้องรักษาค่า search และ category filter
- [ ] ระบบต้องไม่ดึงสินค้าทั้งตารางมาแบ่งหน้าใน application memory

---

## US-06 — เรียงลำดับตามราคา (Optional / Stretch)

**ในฐานะ** ผู้เยี่ยมชม  
**ฉันต้องการ** เรียงสินค้าตามราคา  
**เพื่อ** เปรียบเทียบราคาจากน้อยไปมากหรือมากไปน้อยได้ง่าย

### Acceptance Criteria

- [ ] รองรับการเรียงราคาจากน้อยไปมาก
- [ ] รองรับการเรียงราคาจากมากไปน้อย
- [ ] ค่า sort สะท้อนใน URL
- [ ] sort ทำงานร่วมกับ search, category filter และ pagination
